# Fusion CAM Assistant

Fusion 360 の切削データ作成（CNC: KitMill RZ420）を自動化するアドイン。
ボディを解析して輪郭・穴・ポケットを自動分類し、CAM テンプレートから切削データを一括生成、
NC ファイルの出力までを数クリックにする。

部内 wiki の手順書「切削データ作成のススメ」の作業のうち、毎回手でやっていた
「セットアップ作成 → 図形の選択とテンプレート適用 → 加工順の整理 → ポスト処理」を自動化したもの。

## できること

製造ワークスペースの「工具」タブに「**CAM アシスタント**」パネルが追加される。

| コマンド | 内容 |
|---|---|
| **切削データ自動作成** | ボディを解析して 外郭／内郭／穴あけ／ざぐり／内郭ポケット／取り残し／面取り を自動分類し、確認ダイアログで内容を確認してからテンプレートで一括生成。生成されるセットアップ「自動CAM」は再実行で置き換わる（増殖しない）。原点（WCS）は**ストック上面・左下手前の角**に自動設定される（機械側のゼロ合わせをこの角に合わせる）。ツールパス生成が2件以上のときはプログレスダイアログを表示する（フリーズに見える問題の対策） |
| **NC一括出力** | 同一工具ごとにまとめ、切削順の名前（`1_flat3` 等）で NC ファイルを一括ポスト。**加工指示書.txt**（実行順・工具・操作の一覧）も出力。ポスト前にツールパスを自動で再生成するので「要更新」のまま出力される事故がない。出力ファイルが2件以上のときはプログレスダイアログを表示する |
| **自動配置** | 標準の整列（Arrange）フィーチャを使って、トップレベルコンポーネントを板（原点基準）に自動で詰め直す。**デザイン・製造どちらの「CAM アシスタント」パネルからも実行可能**。実行時のダイアログで枠の幅/奥行・部品間隔・枠からの余白・ソルバー（自動／トゥルーシェイプ／矩形）・回転（任意／90°単位／180°のみ／回転なし）を指定でき、初期値は config.json の値。上下反転と枠外は自動チェックして結果に表示。「完了後に製造ワークスペースへ切替」にチェックすれば完了後に製造ワークスペースへ自動で切り替わる。実行後は「切削データ自動作成」を再実行すること。**Fusion 2025年1月以降が必要** |
| **配置チェック** | 枠への収まり・部品間 8mm 間隔・高さ揃えを検査。配置ガイドの枠スケッチも作成可能。デザイン側の「CAM アシスタント」パネルからも実行できる |
| **設定** | **所有している工具**と優先テンプレセット（標準/DLC）を選択。持っていない工具のテンプレは自動割当・選択肢から除外される。設定は config.local.json に保存され、アップデートで消えない |
| **バージョン情報** | バージョン・更新の有無・テンプレ/工具・ポストの状態を表示（不具合報告にはこれのスクショを添付） |

生成後の**シミュレーション確認（ボトム高さ・ピッチ・衝突）は従来どおり必ず自分の目で行うこと**。
アドインはそこを省略させない前提で作ってある。

## テンプレートセット

2つのセットを同梱しており、「設定」コマンドで使い分ける。

- **標準セット**（`templates/Φ*/`）— サークル標準の単一テンプレファイル。
  命名規約 `[Φ<工具径>]<種別>`（例: `[Φ1.5]Φ2.0 穴あけ`）。この規約で追加すれば自動認識される
- **DLC セット**（`templates/inventor/`）— DLC コート工具用の高速加工テンプレ（Inventor 由来の操作セット形式）。
  ボア（ヘリカル穴あけ: M3下穴Φ2.5・Φ3、ざぐりΦ4・Φ6）、負荷制御、面取り、内外輪郭を含む。
  説明文と種別の対応は config.json の `dlc_name_rules` で編集できる。
  `flat (1).hsmlib` は DLC 工具ライブラリ（参考。テンプレに工具が埋め込まれているため必須ではない）

### 自動分類のルール（概要）

- 穴あけ: Φ2.0・Φ2.6 → 低速輪郭（標準）、Φ2.5・Φ3.0 → ボア（DLC）
- 内郭: 開口幅から入る工具を自動選定。**開口が広く中の「島」が飛びそうな場合は、
  ダイアログのドロップダウンで「くり抜き」系（内郭ポケット/負荷制御）に切り替えられる**
- 取り残し加工: 工具が入らない凹み（尖った隅・小Rフィレット）を内郭・外郭の両方で検出して提案。
  空のツールパスになった操作は自動削除される
- 面取り: 上端エッジ全周を対象に**既定OFF**の行として提案（必要なときだけチェック）
- 加工順: ポケット/ざぐり → 穴あけ → 内郭 → 取り残し → 面取り → **外郭（最後）**

## 動作要件

- Fusion 360（2023年10月以降のバージョン。**自動配置コマンドのみ 2025年1月以降が必要**で、古い場合はその旨を案内して終了する）
- ポストライブラリ（ローカル）に KitMill 用ポスト **originalmind.cps** が登録済みであること
  - 未登録なら [Autodesk のポストライブラリ](https://cam.autodesk.com/hsmposts?p=originalmind)
    からダウンロードし、製造 → 管理 → ポストライブラリ → マイポスト → ローカル にインポートする
- Windows（インストールスクリプトが PowerShell 用のため。Mac は手動でリンクを作れば動くはず・未検証）

## インストール

### ZIP でインストール（かんたん・推奨）

1. GitHub の「Code」→「Download ZIP」でダウンロードして展開する
2. 展開したフォルダを**消さない場所**（ドキュメント配下など）に置く
   （アドインはこのフォルダを参照し続けるため、Downloads に置いたまま消すと壊れる）
3. フォルダ内の **install.bat をダブルクリック**
4. Fusion で：ユーティリティ → アドイン（Shift+S）→ FusionCamAssistant を「実行」
   （「起動時に実行」にチェック推奨）

### ZIP でアップデート

1. Fusion を閉じる（起動したままでも大抵動くが、確実なのは閉じること）
2. GitHub から新しい ZIP をダウンロードして展開する
3. **今使っているフォルダの中身を全部削除**し、新しい ZIP の中身を**同じフォルダに**コピーする
   - フォルダの場所と名前は変えないこと（アドインへのリンクが切れるため）
   - config.local.json（個人設定）だけは残すか、退避して戻す
4. Fusion を起動する（install.bat の再実行は不要）

新バージョンが出ると起動時に通知が表示される。

**v0.2 以前（アドイン名 QuhpCamAssistant）からの移行**: フォルダを更新したあと
**install.bat を一度実行**する（旧リンクは自動で削除され、新しい名前で登録し直される）。

### アンインストール

1. Fusion を閉じる（起動中なら Shift+S → FusionCamAssistant を停止し「起動時に実行」のチェックを外す）
2. フォルダ内の **uninstall.bat をダブルクリック** — Fusion への登録（リンク）と一時ファイルが消える
3. 不要ならフォルダごと削除する（テンプレートや個人設定もここに入っている）

補足:
- 生成済みの切削データ（セットアップ「自動CAM」等）は各ドキュメント内に残る。不要ならブラウザから削除する
- uninstall.bat が使えない場合の手動削除: `%APPDATA%\Autodesk\Autodesk Fusion 360\API\AddIns\FusionCamAssistant`
  を削除する（**リンクなので中身は消えない**。ZIP 展開先を直接コピーした場合は実体なので注意）

### git でインストール（更新が楽）

```powershell
git clone https://github.com/masa2429/fusion-cam-assistant.git
cd fusion-cam-assistant
.\install.bat
```

更新は `git pull` するだけ（install.bat の再実行は不要）。

## 設定

ほとんどの設定は「**設定**」コマンド（所有工具・優先セット）で足りる。
細かい値を変えたいときは `FusionCamAssistant/config.local.json` に config.json と同じキーを書く
（上書きマージされ、アップデートで消えない）。**config.json 本体は編集しないこと。**

| キー | 意味 | 既定値 |
|---|---|---|
| `template_dir` | テンプレートフォルダ。空ならリポジトリ内の `templates/` | `""` |
| `post_name_hint` | ポストライブラリから探すポスト名 | `originalmind` |
| `stock.width_mm` / `depth_mm` | 配置枠・ストックの基準サイズ（**暫定値**） | 280 × 280 |
| `part_gap_mm` | 部品間隔の下限 | 8.0 |
| `stock_side_margin_mm` | ストック側面の余白（外郭パスの逃げ場） | 5.0 |
| `force_safe_linking` | ポケット/負荷制御の進入・リンクを安全化（ストック外からの水平進入と下がったままの移動を禁止）。**実部材は仮想ストックより大きいため必須** | true |
| `hole_rules` / `dlc_name_rules` | 穴径・DLCテンプレの対応ルール | Φ2.0, Φ2.6 ほか |
| `preferred_tools` | 加工種別ごとの工具径の優先順位 | 内郭 [4,3,2,1.5] など |

## 既知の制限・未検証事項

- ざぐりのボア（Φ4/Φ6ザグリ）は実データでの検証がまだ（貫通穴のボア・面取りは検証済み）
- ルール外の円形穴（ボール盤で開ける穴）は「未割り当て」として表示のみ（従来どおりスポットドリルで印つけ）
- 取り残しの要否判定は安全側（不要な提案は空パスとして自動削除される）
- 配置チェックの間隔・重なり判定はバウンディングボックスによる目安
- 自動配置のトゥルーシェイプ（実形状）詰めのあとは、「配置チェック」の間隔警告（bbox 目安）が出ることがある（実形状で 8mm 確保済みなら問題なし）
- 自動配置はルート直下のボディを動かせない（「ボディからコンポーネントを作成」してから使う）
- ポケット加工はストックに突っ込まないように調整はしていますが，パーツによってはうまくいかないことがありそうです．自分で再生成してください．
- あくまでこのツールは補助です．

## トラブルシューティング

- まず「**バージョン情報**」コマンドを実行して状態を確認する（テンプレ件数・ポスト解決・更新有無）
- 詳しいログは Fusion のテキストコマンドウィンドウ、または `%TEMP%\fusioncam.log`
- **アドイン一覧に FusionCamAssistant が2つ出る** — 手動追加とリンクの重複。どちらか片方だけを実行する
- **「テンプレートが見つかりません」** — 展開フォルダを移動/削除していないか確認
- **ポストが見つからない** — ポストライブラリ（ローカル）に originalmind.cps を登録する
- 不具合は Issue へ。バージョン情報のスクショとログがあると早い

## 開発者向け

- ブランチ運用: `main` は安定版（ZIP・更新通知の参照先）、開発は `dev` で行いリリース時にマージ＋version 更新
- [CLAUDE.md](CLAUDE.md) — 開発ルール・ミスの記録（AI アシスタント用の指示書を兼ねる）
- [docs/api-notes.md](docs/api-notes.md) — Fusion CAM API の調査結果（廃止 API・実機確認済みパラメータ名）。**CAM API を触る前に必読**
- [tools/DumpParameters](tools/DumpParameters) — セットアップ・操作の全パラメータをダンプする調査用スクリプト
- パネルボタンのアイコン: `tools/dev/make_icons.py` を編集して `py -3 tools/dev/make_icons.py` を実行すると
  （要 Pillow）`FusionCamAssistant/resources/<コマンドID>/` に 16/32/64px の PNG が生成される。
  生成物はコミットする。フォルダ名はコマンド ID と一致させること
