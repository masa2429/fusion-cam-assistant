# QUHP CAM Assistant

QUHP の切削データ作成（Fusion 360 → KitMill RZ420）を自動化する Fusion 360 アドイン。

手順書（[docs/切削データquhp.md](docs/切削データquhp.md)）の作業のうち、毎回手でやっていた
「セットアップ作成 → 輪郭・穴の選択とテンプレート適用 → 加工順の整理 → ポスト処理」を
ボタン数回にする。

## できること

製造ワークスペースの「工具」タブに「QUHP CAM」パネルが追加される。

| コマンド | 内容 |
|---|---|
| **配置チェック** | 280×280 枠への収まり・部品間 8mm 間隔・高さ揃えを検査。配置ガイドの枠スケッチも作成可能 |
| **切削データ自動作成** | ボディを解析して 外郭／内郭／穴あけ（Φ2.0・Φ2.6）／ざぐり／内郭ポケット を自動分類し、確認ダイアログで内容を確認してから、サークル標準テンプレートで切削データを一括生成。加工順は「ポケット→穴→内郭→取り残し→**外郭（最後）**」を厳守 |
| **NC一括出力** | 同一工具ごとにまとめ、切削順の名前（`1_flat3`, `2_flat1.5`…）で NC ファイルを一括ポスト。番号順に CNC で実行すればよい |

生成後の**シミュレーション確認（ボトム高さ・ピッチ・衝突）は従来どおり必ず自分の目で行うこと**。
アドインはそこを省略させない前提で作ってある。

## 動作要件

- Fusion 360（2023年10月以降のバージョン）
- ポストライブラリ（ローカル）に KitMill 用ポスト **originalmind.cps** が登録済みであること
  - 未登録なら [post/originalmind.cps](post/originalmind.cps) を 製造 → 管理 → ポストライブラリ → マイポスト → ローカル にインポートする
- Windows（インストールスクリプトが PowerShell 用のため。Mac は手動でリンクを作れば動くはず・未検証）

## インストール

```powershell
git clone https://github.com/masa2429/quhp-cam-assistant.git
cd fusion-addin
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

（git を使わない場合は Download ZIP → 展開 → install.ps1 を実行でもよい。
ただしテンプレート更新を受け取れるので clone を推奨）

その後 Fusion で：ユーティリティ → アドイン（Shift+S）→ QuhpCamAssistant を「実行」
（「起動時に実行」にチェック推奨）。

## 使い方（標準フロー）

1. 手順書どおりにパーツを配置（280×280、間隔 8mm、高さ揃え）
2. **配置チェック** で確認
3. **切削データ自動作成** → 確認ダイアログで分類結果をチェック → 生成
   - 行のチェックを外すとその加工はスキップされる
   - ドロップダウンで工具径違いのテンプレートに変更できる
   - 「QUHP 自動セットアップ」は再実行すると置き換えられる（増殖しない）
4. **シミュレーションで全データを確認**（必須）
5. **NC一括出力** → 出力先（USB メモリ等）を選ぶ → 番号順に加工

## 設定（QuhpCamAssistant/config.json）

| キー | 意味 | 既定値 |
|---|---|---|
| `template_dir` | テンプレートフォルダ。空ならリポジトリ内の `templates/` を自動使用 | `""` |
| `post_name_hint` | ポストライブラリから探すポスト名 | `originalmind` |
| `stock.width_mm` / `depth_mm` | 配置枠・ストックの基準サイズ | 280 × 280 |
| `part_gap_mm` | 部品間隔の下限 | 8.0 |
| `hole_rules` | 穴径→穴あけテンプレの対応 | Φ2.0, Φ2.6 |
| `preferred_tools` | 加工種別ごとの工具径の優先順位 | 外郭 [3,2] など |

テンプレートは `templates/Φ*/` にあり、ファイル名規約は `[Φ<工具径>]<種別>`。
この規約に従って .f3dhsm-template を追加すれば自動で認識される（**既存テンプレは書き換えない**）。

## 既知の制限・未検証事項

- ざぐり・内郭ポケット（pocket2d 系）は実データでの検証がまだ（輪郭・穴・外郭は検証済み）
- ルール外の円形穴（ボール盤で開ける穴）は自動化されない。ダイアログに「未割り当て」として表示されるので、従来どおりスポットドリルで印つけする
- 「取り残し加工」の要否判定は安全側（不要な空パスを提案することがある）。ダイアログでチェックを外すか、生成後に無効化してよい
- 配置チェックの間隔・重なり判定はバウンディングボックスによる目安

## トラブルシューティング

- **アドイン一覧に QuhpCamAssistant が2つ出る** — 「＋」で手動追加した登録とジャンクションが重複している。どちらか片方だけを実行する（同時実行しない）
- **「テンプレートが見つかりません」** — config.json の `template_dir` を確認。空ならリポジトリの `templates/` が使われる
- **ポストが見つからない** — ポストライブラリ（ローカル）に originalmind.cps を登録する
- 不具合を見つけたら Issue へ。エラーダイアログの全文とスクリーンショットがあると早い
